# Azure VM上でのLAMP環境構築手順書

Azure for Studentsアカウントを使い、仮想マシン(VM)を作成してLinux + Apache + MySQL + PHP(LAMP)環境を構築し、簡単なWebページを公開するまでの手順をまとめる。AWSでのEC2構築(`aws-lamp-setup-guide`)と対になる、Azure版の手順書。

## 1. 事前準備

### 1.1 Azure for Studentsアカウントの作成

- 大学の学籍確認メールアドレスでサインアップ
- クレジットカード登録は不要
- $100分のクレジットが付与され、有効期限は365日

**チェックポイント:** Azureポータルの「Education」概要画面で「利用可能なクレジット: $100個中$100個」と表示されていることを確認する。

## 2. 仮想マシン(VM)の作成

### 2.1 リソースグループの作成

VMやネットワークなど、関連リソースをまとめるための「リソースグループ」を新規作成する(例: `rg-azure-handson`)。AWSにはない概念で、後片付けの際にグループごと一括削除できる利点がある。

### 2.2 基本設定

- 仮想マシン名: 任意(例: `azure-web-server`)
- リージョン: Japan East(東日本)
- イメージ: **Ubuntu Server 24.04 LTS** を明示的に選択する(既定のままだとWindows系イメージになる場合があるため要確認)

### 2.3 サイズ選択における注意

無料枠対象の「B1s」を選択しようとしたが、学生サブスクリプションでは日本東を含む複数リージョンでBシリーズのクォータが0になっており、選択できないケースがあった(エラー: `NotAvailableForSubscription`)。

**対処:** 有料サイズ(例: Standard_D2s_v3、約0.129USD/時間)を使用し、検証後すみやかに削除することでコストを数十円程度に抑える。

### 2.4 認証設定

- 認証の種類: SSH公開キー
- 「新しいキーの組の生成」を選択し、作成完了時に秘密鍵(.pem)を必ずダウンロードする

### 2.5 受信ポート

作成時点ではSSH(22)のみを許可し、詳細な制限は後述のNSG編集で行う。

**チェックポイント:** 「確認と作成」画面で価格(USD/時間)とサブスクリプション名を確認し、「作成」をクリックする。秘密キーのダウンロードダイアログが出たら、必ずダウンロードしてから続行する。

## 3. ネットワークセキュリティグループ(NSG)の設定

VM作成時に自動生成されるNSG(例: `azure-web-server-nsg`)を編集し、AWSのセキュリティグループと同様のアクセス制限を行う。

### 3.1 SSHルールの制限

VMの「ネットワーク」→「ネットワーク設定」から、SSH(ポート22)の受信規則を開き、「ソース」を「Any」から「IPアドレス」に変更、自分の現在のグローバルIPアドレスを`/32`付きで入力して保存する。

### 3.2 HTTPルールの追加

「ポートルールの作成」から新しい受信規則を追加する。

- サービス: HTTP(自動的にポート80が設定される)
- ソース: Any(誕生日Webサーバーとして公開するため)
- 名前: `HTTP`

**チェックポイント:** ルール一覧に、SSH(自分のIPのみ)とHTTP(Any)の2つが「Allow」状態で表示されていることを確認する。

## 4. SSH接続

ダウンロードした秘密鍵を使い、PowerShellから接続する。

```powershell
cd ~\Downloads
ssh -i azure-web-server_key.pem azureuser@<パブリックIPアドレス>
```

初回接続時はホスト鍵の確認(`yes`と入力)が必要。接続後、`azureuser@azure-web-server:~$` のプロンプトが表示されれば成功。

**注意:** 秘密鍵ファイルはブラウザのダウンロード先(通常は`ダウンロード`フォルダ)に保存される。OneDrive同期デスクトップとローカルの`Desktop`フォルダが異なる場合があるため、保存先を必ず確認する。

## 5. LAMP環境の構築

### 5.1 パッケージのインストール

```bash
sudo apt update
sudo apt install -y apache2 php libapache2-mod-php php-mysql mysql-server
```

### 5.2 MySQLユーザーの作成

```bash
sudo mysql
```

MySQLプロンプト内で以下を実行する。

```sql
CREATE USER 'azureadmin'@'localhost' IDENTIFIED BY '(任意のパスワード)';
GRANT ALL PRIVILEGES ON *.* TO 'azureadmin'@'localhost' WITH GRANT OPTION;
FLUSH PRIVILEGES;
EXIT;
```

### 5.3 動作確認

ブラウザで `http://<パブリックIPアドレス>/` にアクセスし、「Apache2 Ubuntu Default Page」が表示されることを確認する。

PHPとMySQL連携の動作確認として、一時的に`info.php`を作成して`phpinfo()`の出力を確認してもよい。

```bash
echo "<?php phpinfo(); ?>" | sudo tee /var/www/html/info.php
```

**注意:** `phpinfo()`は内部構成情報を外部に公開してしまうため、確認後は必ず削除する(`sudo rm /var/www/html/info.php`)。

## 6. Webページの公開

`/var/www/html/index.html`をオリジナルの内容で上書きし、独自のWebページを公開する。

```bash
sudo tee /var/www/html/index.html > /dev/null << 'EOF'
<!DOCTYPE html>
...(任意のHTML)...
EOF
```

ブラウザで再度パブリックIPにアクセスし、作成したページが表示されることを確認する。

## 7. コスト確認

Azureポータルの「サブスクリプション」→「Azure for Students」の一覧で、「現在のコスト」を確認できる。ハンズオン中はここが数十円〜1ドル未満で推移していることを都度確認するとよい。

## 8. 後片付け(リソースグループの削除)

作業が終わったら、個々のリソースを消すのではなく、リソースグループごと削除するのが最も確実で早い。

1. 「リソース グループ」からハンズオン用のグループ(例: `rg-azure-handson`)を開く
2. 「リソース グループの削除」をクリック
3. 削除対象リソース一覧(VM、パブリックIP、NSG、NIC、SSHキー、ディスク、VNetなど)を確認する
4. 確認欄にリソースグループ名を正確に入力し、「削除」を実行する

**チェックポイント:** 削除後、サブスクリプション概要の「現在のコスト」が `0.00` のままであることを確認し、課金対象リソースが残っていないことを確認する。

## 9. AWSとの比較を通じた学び

- Azureの「リソースグループ」は、AWSにはない一括管理・一括削除の仕組みで、後片付けの手間を大きく減らせる
- クォータ(利用可能量)はサブスクリプションとリージョンの組み合わせごとに個別に設定されており、無料枠のサイズが常に使えるとは限らない。エラーメッセージ(`NotAvailableForSubscription`)を正しく読み取り、代替手段(有料サイズ+短時間利用)を選べる判断力が求められる
- Entra ID(組織のID管理)とサブスクリプション(課金・リソース管理)は権限体系が分離されており、大学組織のような既存テナントに参加する形でクラウドを使う場合は、個人の権限範囲を正しく理解しておく必要がある
